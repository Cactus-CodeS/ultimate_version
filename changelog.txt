1.2.0 - 2026-04-10
------------------

  ## rsg-multijob Compatibility
  + Full integration with rsg-multijob resource
  + /addjob command — assign any Job Creator job to a player
  + Automatic job sync between cactus_ultimate and rsg-multijob
  + myJobs callback rewritten for proper multijob support
  + GetFreshJobs() pattern for reliable cross-resource data
  + Install helper included (install/forMultijob.lua)

  ## Craft Integration in Job Creator (CACTUS_CRAFT REQUIRE)
  + Per-job craft stations, recipes & categories — configured directly from the Job Creator UI
  + New "Craft Settings" page — global craft configuration from the Job Creator dashboard
  + Live config source indicator (Database / Live / Defaults)
  + Craft section & settings page auto-hidden when cactus_craft is not installed
  + Item image map — correct item images pulled from framework (RSGCore.Shared.Items)
  + Full NUI: buildCrafting(), collectFormIntoEditing, translations (100+ keys FR/EN)
  + DB migration: craftStations, craftRecipes, craftCategories JSON columns in jobcreator_jobs
  + DB table: craft_global_settings (single-row JSON storage)

  ## Hot-Reload System
  + Save craft settings → applied instantly in-game, no restart needed
  + Save per-job craft data → stations, recipes & categories updated live via ReloadCraftData
  + Global settings (notification system, XP, keys, images...) hot-reloaded to all players
  + ReloadGlobalSettings export triggers cactus_craft live config update

  ## Auto Framework Detection
  + Framework auto-detected from running resources (rsg-core / vorp_core) — zero config needed
  + Framework field locked (read-only) in Craft Settings UI
  + Notification system remains fully configurable by the user (select dropdown)
  + Detected framework injected into GetAllCraftConfigs and craft settings on load/save

  * Fixed missing NUI toast notifications (showNotif → showToast)
  * Fixed craft data not loading in job editor (LoadAllJobsForCreator missing 3 craft columns)
  * Fixed InventoryImgPath not propagating to client (added to DynamicGlobalSettings broadcast)
  * Fixed item images using wrong filenames (imageMap system from RSGCore.Shared.Items)
  * Fixed resource name mismatch (rsg_inventory → rsg-inventory)
