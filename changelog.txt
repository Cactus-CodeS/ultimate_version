1.1.0 - 2026-03-23
------------------

  ## Community Hub — Job Import & Sharing
  + Browse jobs published by other server owners via the Community Hub
  + One-click import — install any community job straight into your database
  + Share your own custom job creations with the community
  + Discord profile integration (avatar & username displayed automatically)
  + Community Discord webhook logging for import & share actions
  + GitHub-based backend — all jobs are version-controlled

  ## Action History
  + Full company action history — every action logged and viewable by managers
  + Tracks: hire, fire, promote, demote, deposits, withdrawals, bonuses, salary payments
  + Per-action details: actor, target, old value, new value, amount, timestamp
  + Permission-based access: `canViewHistory` and `canViewFullHistory`
  + Filterable and sortable in the NUI interface

  ## Discord Webhooks (Enhanced)
  + Per-job Discord webhook configuration (set via boss menu)
  + Webhook logging for harvest actions (items harvested, zone, quantity)
  + Webhook logging for sales (items sold, revenue split company/player)
  + Webhook logging for shop purchases (player-paid or company-paid)
  + Webhook logging for salary payments
  + Webhook logging for deposits & withdrawals
  + Color-coded embeds per action type (green = harvest/hire, red = fire, gold = sell, etc.)

  ## RSG-Core & VORP Optimization
  + Automatic job synchronization at startup — jobs auto-registered in framework
  + RSG-Core: direct integration with RSGCore.Shared.Jobs registry
  + RSG-Core: native job management (Bridge.JobExists, Bridge.AddJobToCore, Bridge.UpdateJobInCore)
  + VORP Core: custom in-memory JobsRegistry (VORP has no native jobs system)
  + VORP Core: WrapVORPPlayer() — normalizes VORP character data to unified format
  + VORP Core: custom duty system implementation (VORP has no native on-duty support)
  + Grade synchronization from config to database (SyncGradesFromConfig)
  + ConvertJobToCoreFormat() — converts Config.Companies to framework-native format
  + Automatic framework detection — zero config change needed between RSG & VORP

  ## Bug Fixes & Improvements
  * Optimized SQL queries with proper indexing for history and community tables
  * Improved grade caching system for better performance
  * Added `webhook` column to businesses table for per-job webhook URLs
  * Install helpers updated for RSG-Core and multijob compatibility
